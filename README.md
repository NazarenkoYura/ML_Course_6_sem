# Online Shoppers Purchasing Intention Prediction

## Описание проекта

Это курсовая работа по машинному обучению, целью которой является **предсказание намерения покупателя совершить покупку** на основе поведения во время сеанса просмотра онлайн-магазина.

**Датасет:** Online Shoppers Purchasing Intention Dataset  
**Размер:** 12,330 сеансов, 18 признаков  
**Целевая переменная:** Revenue (бинарная классификация)

## Описание данных

### Признаки (18):

**Поведенческие метрики:**
- `Administrative` - количество посещений административных страниц
- `Administrative_Duration` - время, проведенное на административных страницах
- `Informational` - количество информационных страниц
- `Informational_Duration` - время на информационных страницах
- `ProductRelated` - количество посещений страниц товаров
- `ProductRelated_Duration` - время на страницах товаров
- `BounceRates` - процент отказов (bounce rate) по сеансу
- `ExitRates` - процент выходов (exit rate) по странице
- `PageValues` - средняя стоимость страницы в сеансе

**Временные параметры:**
- `SpecialDay` - близость сеанса к специальному дню (праздник)
- `Month` - месяц сеанса

**Технические параметры:**
- `OperatingSystems` - ОС посетителя
- `Browser` - браузер посетителя
- `Region` - регион пользователя
- `TrafficType` - тип трафика

**Описательные:**
- `VisitorType` - тип посетителя (Новый, Возвращающийся, Прочие)
- `Weekend` - приходится ли сеанс на выходной (бинарная)

### Целевая переменная:
- `Revenue` - совершена ли покупка (TRUE/FALSE)
  - **Дисбаланс классов:** ~84% без покупок, ~16% с покупками

## Основной процесс анализа

### 1. **Загрузка и валидация данных**
   - Чтение CSV файла
   - Проверка пропусков и дубликатов
   - Валидация типов данных

### 2. **Разведочный анализ (EDA)**
   - Анализ распределений признаков
   - Построение корреляционной матрицы
   - Изучение сезонности (тренды по месяцам)
   - Анализ конверсии по типам посетителей
   - Выявление выбросов

### 3. **Предобработка и Feature Engineering**
   - One-Hot кодирование категориальных признаков (Month, VisitorType)
   - Масштабирование числовых признаков (StandardScaler)
   - Стратифицированное разделение данных:
     - Train: 60%
     - Validation: 20%
     - Test: 20%

### 4. **Моделирование**
   Обучены и сравнены следующие модели:
   
   - **Logistic Regression** - базовая линейная модель
   - **Random Forest** - ансамбль деревьев решений
   - **XGBoost** - градиентный бустинг
   - **MLP (Neural Network)** - многослойный перцептрон (TensorFlow/Keras)

   **Техники оптимизации:**
   - Компенсация дисбаланса классов (class_weight, scale_pos_weight)
   - Перебор гиперпараметров (RandomizedSearchCV)
   - 5-кратная кросс-валидация
   - Ранняя остановка (EarlyStopping)

### 5. **Оценка моделей**
   **Метрики:**
   - ROC-AUC
   - PR-AUC (Precision-Recall)
   - F1-Score
   - Precision и Recall
   - Confusion Matrix

### 6. **Интерпретация результатов**
   - Глобальная интерпретация модели (SHAP)
   - Локальная интерпретация предсказаний
   - Анализ важности признаков

---

## Ключевые наблюдения

- **PageValues** имеет сильную положительную корреляцию с покупкой
- **BounceRates** и **ExitRates** отрицательно связаны с конверсией
- Длительность и количество просмотров **ProductRelated** - важные предикторы
- Значительный **дисбаланс классов** требует применения специальных техник
- Конверсия варьируется по месяцам (сезонность)
- **Возвращающиеся посетители** (Returning_Visitor) имеют более высокую вероятность покупки
